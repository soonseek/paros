1. 제품 비전 (Vision)

본 시스템은 단순히 실무를 “대신 수행”하는 AI가 아니라,
실무 결과와 AI 결과의 차이를 지속적으로 학습·개선하는 구조를 갖는다.

이를 위해 본 제품은 다음 두 가지 코파일럿을 포함한다:

실무 코파일럿 (Work Copilot)
→ 실무자와 함께 일한다

QA 코파일럿 (Quality Assurance Copilot)
→ AI 결과를 사람의 실제 작업 결과와 비교해 개선점을 도출한다

6. QA 코파일럿 (QA Copilot) 설계
6.1 QA 코파일럿의 역할

QA 코파일럿은 판정자가 아니라 분석자다.

다음 질문에 답한다:

AI가 틀렸는가?

아니면 표현만 다른가?

실무 리스크로 이어질 차이인가?

개선한다면 어디를 고쳐야 효과가 큰가?

6.2 비교 기준 (Evaluation Dimensions)

QA 코파일럿은 최소 다음 축으로 비교한다:

1️⃣ 구조 차이

항목 누락

항목 순서 문제

필수 섹션 부재

2️⃣ 수치 차이

합계 불일치

기간 오류

거래 누락/중복

3️⃣ 논리 차이

자금 흐름 설명 누락

원인–결과 연결 부족

보정명령 요구사항 미충족

4️⃣ 법적 위계 사용 차이

법률 근거 누락

준칙/관행 혼용

근거 레벨 불명확

6.3 “의미 있는 개선” 판단 기준

QA 코파일럿은 모든 차이를 리포트하지 않는다.

다음 조건 중 하나라도 충족 시 “개선 필요”로 판단:

❗ 법적 근거 누락/오인

❗ 보정명령 대응 실패 가능성

❗ 자금 흐름 오해 가능성

❗ 실무자가 추가 작업을 해야만 제출 가능

7. QA 리포트 출력 포맷
7.1 리포트 예시
QAReport {
  project_id,
  target: "DocumentGeneration" | "FundFlow",
  severity: "LOW" | "MEDIUM" | "HIGH",
  issues: [
    {
      type: "MissingLegalBasis",
      description: "채무자회생법 제○○조 인용 누락",
      suggested_fix: "LAW RAG 참조 단계 강화 필요",
      affected_agent: "DraftingAgent"
    },
    {
      type: "FlowExplanationGap",
      description: "계좌 A → B 이동에 대한 설명 부재",
      suggested_fix: "Flow Agent에서 narrative 생성 강화",
      affected_agent: "TransactionFlowAgent"
    }
  ]
}
